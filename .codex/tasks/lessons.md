# Lessons

## 2026-03-22

1. What went wrong
   I proposed "best-effort parity" as a possible v1 boundary before confirming whether strict external API/interface parity was a core product promise.
2. Preventive rule
   For any porting or migration product, explicitly confirm the required parity target before treating imperfect parity as an acceptable default.
3. Trigger signal to catch it earlier
   The user describes the product as a porting framework or migration workflow where external APIs, interfaces, or behavioral equivalence are central to success.
