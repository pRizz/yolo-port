Normalize repository intake across `yolo-port`, `yolo-port bootstrap`, and `yolo-port <github-url>` so the CLI can inspect before cloning, keep subcommands visible, and stop safely on dirty local repos.

Purpose: Phase 2 starts by making the entry path context-aware. Users should not need to memorize subcommands just to begin a port, but the visible command surface still needs to stay explicit for power users and future phases.
Output: A shared intake descriptor model, structured git inspection boundary, commandless or URL-driven entry routing, and smoke coverage proving remote and local intake start correctly.
</objective>
