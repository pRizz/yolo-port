## Task 2
<task type="auto">
  <name>Task 2: Implement the Node launcher and Bun entrypoint boundary</name>
  <files>bin/yolo-port.js, src/cli/main.ts, src/cli/router.ts</files>
  <action>Build a Node-based launcher that can always start under npm-installed Node. Keep its responsibility narrow: resolve the package root, render native help/version output when those are requested, and hand off to Bun only when Bun is already available. Do not embed the real bootstrap flow in the Node launcher; Phase 1 needs this shell mainly to survive Bun-less machines and to keep the Bun handoff strategy explicit and debuggable.</action>
  <verify>`node bin/yolo-port.js --help` prints usage successfully from the repo root.</verify>
  <done>The CLI starts under Node, help works without Bun preinstalled, and Bun handoff targets `src/cli/main.ts` instead of a Bun shebang in npm's `bin` field.</done>
</task>
